15. **Volatility targeting (continuous leverage scaling) — new mechanism, scoped 2026-07-09, not yet built.** Every strategy in this repo uses discrete leverage (0x/1x/2x) gated by the trend signal. Untested alternative: within a confirmed-bull regime, scale leverage continuously against trailing realized volatility instead of holding it fixed at 2x — `target_leverage = clip(target_vol / realized_vol, floor, cap)`. Mechanically different from the rejected `tools.regime_probability` (which tried to predict *direction* via logistic regression, item covered in [quant_toolbox.md](quant_toolbox.md)): vol-targeting doesn't predict anything, it risk-manages an *already-confirmed* trend (Moreira & Muir "volatility-managed portfolios" literature).
    - **Scope:** `core/simulator.py::run` already accepts continuous leverage in `positions` (target leverage is just a float — no simulator change needed). New work is one strategy module: `strategies/vol_target.py::positions(prices, signal, target_vol, window, floor, cap)` that (a) keeps the existing MA-crossover regime gate for bear-exit (unchanged — 1x hold in bear, same as `strategies/momentum.py`), (b) replaces the fixed 2x with `clip(target_vol/realized_vol, floor, cap)` while in the bull regime. Realized vol = rolling `window`-day annualized stdev of daily returns (start at 20d, grid 10/20/63d). `floor = 1.0` (never delever below the "never go below 1x" rule in [methodology.md](methodology.md)); `cap` must stay comfortably under `1/MAINTENANCE_MARGIN ≈ 3.33x` (`core/config.py`) to avoid mechanically forcing margin calls — start the grid at 2.5–3.0x. Backtest tool: a new `tools/vol_target_backtest.py`, or a `--vol-target` flag on `tools.backtest`, reusing `core/metrics.py` as-is.
    - **Method (mandatory, JUDGMENT.md §5 new-finding gate):** walk-forward OOS over the (window, target_vol, cap) grid, same discipline as `tools.optimize`. The significance test needs care: a circular-shift-vs-random-timing null answers "does the trend signal predict direction," which isn't the question here (the trend gate is unchanged from the live strategy). The correct comparison is **matched-average-leverage**: build a fixed-leverage baseline equal to the vol-targeting strategy's *realized average leverage* over the same regimes, then compare Sharpe/Calmar/MaxDD at that matched average exposure — otherwise a strategy that simply delevers on average will look "safer" for free, the same bug class as the sector-rotation leverage-fairness catch (`docs/agents/LESSONS.md`, 2026-07-06).
    - **Acceptance:** OOS table + matched-average-leverage comparison + plausibility check against `docs/agents/JUDGMENT.md` §6 (leveraged-strategy MaxDD range). Verdict vocabulary per JUDGMENT §7.5 applies.
    - **Candidate tickers:** SPMO and GLD first (the live legs — most useful result either way); SMH optional second pass.
